# PRD: ESG 지표산정 자동화 Agent

문서 버전: 0.1
작성일: 2026-04-24  
프로젝트명: ESG 지표산정 자동화 Agent  

## 1. 제품 개요

ESG 지표산정 자동화 Agent는 기업별 ESG/기업건전성 지표 산정 업무를 자동화하는 시스템이다. 핵심 목적은 사람이 장기간 반복하던 자료 수집, 지표 계산, 결과 검증, 보고서 작성 과정을 재현 가능하고 검증 가능한 파이프라인으로 전환하는 것이다.

이 제품은 LLM이 직접 점수를 추측하는 구조가 아니다. 기본 수집과 계산은 규칙 기반 파이프라인이 담당한다. 다만 규칙 기반 수집으로 해결하기 어려운 비정형 자료 탐색, 원천 후보 확인, 누락 근거 보완은 Agent가 수행하고, 이후 정답 검증과 리포트 초안 작성에도 Agent를 활용한다.

핵심 처리 흐름은 다음과 같다.

```text
Rule-Based 수집 및 계산 -> Agent 보완 수집 -> 정답 검증 Agent -> 리포트 작성 Agent
```

## 2. 배경

기존 기업건전성 지표산정 업무는 여러 기업의 공시, 재무, 인증, 제재, 소비자, 환경 관련 자료를 사람이 직접 확인하고 계산하는 방식으로 진행됐다. 이 과정은 약 9개월이 걸릴 정도로 시간이 오래 걸렸고, 담당자별 판단 차이와 반복 검증 비용이 컸다.

전년도 수행에서는 LLM 중심 자동화 방식이 적용됐으나, LLM에 과도하게 의존하면서 요구사항을 충분히 만족하지 못했고 완성에도 이르지 못했다. 현재 팀은 해당 과제를 인계받아, 계산 책임을 Agent가 아니라 명시적 규칙과 검증 가능한 데이터 파이프라인에 두는 방향으로 제품을 재설계한다. Agent는 규칙으로 포착하기 어려운 자료 보완, 검증 설명, 보고서 작성처럼 추론과 문서화가 필요한 영역에 제한적으로 사용한다.

## 3. 문제정의

현재 업무의 핵심 문제는 다음과 같다.

- 수작업 기간이 길고 반복 비용이 높다.
- 같은 평가 기준을 여러 기업과 연도에 일관되게 적용하기 어렵다.
- 최종 점수의 근거 자료와 계산 과정을 추적하기 어렵다.
- 산출 결과가 정답 데이터와 다를 때 오류 원인을 빠르게 찾기 어렵다.
- LLM 단독 의존 방식은 재현성과 신뢰성이 낮다.
- 보고서 작성이 계산 결과와 분리되어 있어 설명 품질이 담당자 역량에 크게 의존한다.

## 4. 제품 목표

1. 기업별 원천 자료를 체계적으로 수집하고 보존한다.
2. 지표 계산 규칙을 코드와 설정으로 관리해 동일한 입력에서 동일한 결과가 나오게 한다.
3. 산출값과 정답값의 차이를 자동으로 비교하고 오류 원인을 추적한다.
4. 최종 점수뿐 아니라 출처, 근거, 예외 처리, 검증 상태를 함께 남긴다.
5. 검증된 산출 결과를 바탕으로 기업별 리포트 초안을 자동 생성한다.

## 5. 핵심 기능

### 5.1 Rule-Based 수집 및 계산

기업별 지표 산정에 필요한 원천 자료를 수집하고, 정규화된 데이터 모델로 변환한 뒤, 지표별 계산 규칙에 따라 결과를 산출한다.

요구사항:

- 회사명, 종목코드, 연도 기준으로 자료를 조회한다.
- DART, KRX, 공공기관 자료, PDF, Excel, 웹 문서 등 다양한 원천을 다룬다.
- 원본 자료와 가공 데이터를 분리 저장한다.
- 지표별 계산 규칙을 명시적으로 관리한다.
- 계산 결과마다 사용된 원천 자료와 중간값을 연결한다.
- 산출 결과를 Excel, CSV, JSON 형태로 내보낼 수 있다.

### 5.2 Agent 보완 수집

규칙 기반 수집으로 찾기 어려운 비정형 자료, 누락 근거, 대체 원천 후보를 Agent가 탐색한다. Agent는 계산값을 직접 결정하지 않고, 후보 원천과 근거를 수집해 rule-based 파이프라인이 다시 검토할 수 있는 구조화된 자료로 넘긴다.

요구사항:

- rule-based 수집 실패 또는 근거 부족 항목을 입력으로 받는다.
- 회사명, 연도, 지표명, 필요한 근거 유형을 기준으로 원천 후보를 탐색한다.
- Agent가 찾은 자료는 URL, 문서명, 발행기관, 수집 시각, 관련 문장/표 위치를 함께 기록한다.
- 수집 후보는 신뢰도와 사용 가능 여부를 표시하되, 최종 계산 반영은 rule-based 검증 단계를 거친다.
- Agent 수집 결과와 기존 rule-based 결과가 충돌할 경우 수기 검토 대상으로 표시한다.

### 5.3 정답 검증 Agent

자동 산출 결과를 정답 데이터와 비교하고, 차이가 발생한 항목의 원인을 분석한다. Agent는 값을 새로 만들어내지 않고, 산출 결과, 정답 데이터, 원천 근거, 계산 trace를 읽어 검증 설명을 생성한다.

요구사항:

- 정답 데이터와 산출 데이터의 항목별 차이를 계산한다.
- 10% 랜덤 샘플 검증을 먼저 수행하고, 기준을 통과한 뒤 전체 데이터 검증으로 확장한다.
- 섹션별 정확도, positive recall, precision, 오류 유형을 기록한다.
- 오차 발생 항목의 원천 자료, parser, 계산 규칙, fallback 경로를 추적한다.
- 수정 우선순위와 재검증 범위를 제안한다.

### 5.4 리포트 작성 Agent

검증된 산출 결과와 근거 자료를 바탕으로 기업별 또는 전체 배치 리포트 초안을 작성한다. 리포트는 단순 점수 나열이 아니라 지표별 근거, 위험 항목, 검증 상태를 설명해야 한다.

요구사항:

- 기업별 ESG/기업건전성 지표 요약을 생성한다.
- 지표별 산정 근거와 출처를 표시한다.
- 정답 검증 결과와 남은 위험 항목을 포함한다.
- 표와 차트 중심의 요약을 지원한다.
- Markdown, HTML, Excel 등으로 확장 가능한 출력 구조를 제공한다.

## 6. 사용자

| 사용자 | 역할 | 필요한 가치 |
| --- | --- | --- |
| 지표 산정 담당자 | 기업별 자료 수집과 산정 결과 검토 | 수작업 감소, 계산 재현성, 오류 추적 |
| 검수자 | 산출 결과와 정답 데이터 비교 | 정확도 지표, 오류 원인, 수정 우선순위 |
| 보고서 작성자 | 산정 결과를 외부 보고 문서로 정리 | 근거가 포함된 일관된 리포트 초안 |
| 의뢰 단체 담당자 | 최종 결과물을 검토하고 활용 | 신뢰 가능한 점수, 근거 자료, 설명 가능성 |

## 7. 제품 범위

### 포함

- 기업별 원천 자료 수집
- Agent 기반 누락 자료 보완 수집
- 원본 자료 저장과 수집 이력 관리
- 지표별 rule-based 계산
- 산출 결과와 정답 데이터 비교
- 오류 원인 분석과 검증 리포트 생성
- 기업별 결과 리포트 초안 생성

### 제외

- Agent가 근거 없이 지표 값을 직접 생성하는 기능
- Agent가 찾은 자료를 검증 없이 계산에 바로 반영하는 기능
- 특정 기업의 정답을 코드에 하드코딩하는 방식
- 정답 파일을 복사해 산출값처럼 사용하는 방식
- 근거 자료 없이 점수를 보정하는 기능

## 8. 시스템 요구사항

### 데이터 수집

- 수집기는 동일 원천을 중복 다운로드하지 않아야 한다.
- 원천 자료는 URL, 수집 시각, 회사, 연도, checksum과 함께 보관해야 한다.
- API 실패 시 대체 원천 탐색 경로를 기록해야 한다.
- rule-based 수집 실패 항목은 Agent 보완 수집 queue로 넘길 수 있어야 한다.
- Agent가 수집한 원천 후보는 계산에 바로 쓰지 않고 구조화 검토 대상으로 저장해야 한다.

### 계산

- 계산 로직은 deterministic해야 한다.
- 지표별 계산 규칙은 테스트 가능해야 한다.
- 계산 결과에는 사용된 입력값, 중간값, rule id가 남아야 한다.

### 검증

- 전체 검증 전에 10% 랜덤 샘플 검증을 수행해야 한다.
- 샘플 검증 기준을 통과하지 못하면 전체 검증으로 확장하지 않는다.
- 오류는 parser 오류, source 오류, mapping 오류, rule 오류, missing data로 분류한다.

### 리포트

- 리포트에는 점수, 근거, 검증 상태, 남은 위험을 포함해야 한다.
- 리포트의 모든 주요 주장에는 source 또는 metric trace가 연결되어야 한다.
- 사람이 최종 검토할 수 있도록 편집 가능한 형태로 출력해야 한다.

## 9. 성공 기준

- 수작업 산정 대비 반복 업무 시간이 유의미하게 줄어든다.
- 동일 입력에 대해 동일한 산출 결과가 재현된다.
- 산출 점수마다 원천 자료와 계산 trace를 확인할 수 있다.
- 10% 샘플 검증 후 전체 데이터 검증으로 확장 가능한 품질 기준을 충족한다.
- 오류 발생 시 어떤 원천, parser, rule에서 문제가 생겼는지 추적할 수 있다.
- Agent 출력은 계산 결과를 대체하지 않고 누락 자료 보완, 검증, 설명 품질을 높이는 데 사용된다.

## 10. 제품 로드맵

### Phase 1. 업무 기준 정리와 데이터 계약 확정

- 평가 지표 목록과 산식 정리
- 원본 Excel/PDF 구조 분석
- 회사, 연도, 지표, source, evidence의 공통 데이터 모델 정의

### Phase 2. Rule-Based 수집/계산 파이프라인 안정화

- DART, KRX, 공공자료, PDF/Excel 수집 경로 정리
- parser와 계산 rule 분리
- 산출 workbook 생성
- 지표별 회귀 테스트 작성

### Phase 3. Agent 보완 수집 체계 구축

- rule-based 수집 실패 항목 queue 정의
- Agent 원천 탐색 결과 schema 정의
- 후보 원천 신뢰도와 수기 검토 상태 관리
- 보완 수집 결과를 rule-based 검토 단계로 되돌리는 흐름 구현

### Phase 4. 정답 검증 체계 구축

- 정답 데이터와 산출 데이터 비교기 구현
- 10% 샘플 검증 프로세스 자동화
- 오류 유형 분류와 재검증 흐름 정리
- 정확도 리포트 생성

### Phase 5. Agent 기반 검증/리포트 고도화

- 정답 검증 Agent가 오류 원인과 수정 우선순위를 설명
- 리포트 작성 Agent가 기업별 요약과 근거 중심 보고서 생성
- chart/table 기반 요약 출력

### Phase 6. 운영화

- 배치 실행 안정화
- 실패 재시도와 audit log 정리
- 전체 데이터 검증 자동화
- 최종 산출물 패키징

| 정답 데이터 기준 불명확 | 정확도 판단 오류 | 지표별 기준과 예외 규칙 문서화 |
