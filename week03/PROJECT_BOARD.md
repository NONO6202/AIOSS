## Board View

GitHub Project view는 Board layout으로 저장했고, `Kanban Status` 기준으로 컬럼을 구성했다.

## Custom Fields

| Field | Type | Options |
| --- | --- | --- |
| Kanban Status | Single select | Backlog, To Do, In Progress, Review, Done |
| Priority | Single select | P0, P1, P2 |
| Story Points | Number | 1, 2, 3, 5 |
| Sprint | Single select | Sprint 1, Sprint 2 |

## 운영 방식

1. 새 작업은 `Backlog`에 등록한다.
2. 이번 스프린트에 착수할 작업은 `To Do`로 이동한다.
3. 작업 시작 시 `In Progress`로 이동한다.
4. 제출 전 검토가 필요하면 `Review`로 이동한다.
5. 검토가 끝난 작업은 `Done`으로 이동한다.
